A C# Web API that lets you manage a collection of video game characters through a RESTful interface.
Built with ASP.NET Core, Entity Framework Core, and SQL Server. API documentation is served via Scalar.

## Project Structure

| File/Folder | Description |
|-------------|-------------|
| `Controllers/` | Handles incoming HTTP requests and returns responses |
| `Services/` | Contains the business logic and database communication |
| `Models/` | Defines the Character entity that maps to the database table |
| `DTOs/` | Data Transfer Objects that shape what goes in and out of the API |
| `Data/` | Contains AppDbContext, the bridge between EF Core and the database |
| `Migrations/` | Auto-generated EF Core migration files |
| `Screenshots/` | Screenshots of the API in action and the database |

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/VideoGameCharacter` | Returns all characters |
| GET | `/api/VideoGameCharacter/{id}` | Returns a character by Id |
| POST | `/api/VideoGameCharacter` | Adds a new character |
| PUT | `/api/VideoGameCharacter/{id}` | Updates an existing character |
| DELETE | `/api/VideoGameCharacter/{id}` | Deletes a character |

## Setup

### Requirements
- .NET 10
- SQL Server
- Entity Framework Core Tools

### Running the project
1. Clone the repository
2. Update the connection string in `appsettings.json` with your SQL Server details:
```json
"DefaultConnection": "Server=YOUR_SERVER;Database=VideoGameCharactersDb;Trusted_Connection=True;TrustServerCertificate=True"
```
3. Run migrations to create the database:
```bash
dotnet ef database update
```
4. Run the project:
```bash
dotnet run
```
5. Open the Scalar UI at:
https://localhost:{port}/scalar/v1

## Notes

- The `Id` is always generated by the database. Never send it in POST or PUT request bodies
- PUT and DELETE return `204 No Content` on success
- POST returns `201 Created` with a `Location` header pointing to the new resource
- Scalar UI is only available in the Development environment
